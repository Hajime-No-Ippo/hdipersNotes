
Q8: React Keys
```javascript
Why is Math.random() bad for list keys?

A) Duplicate key errors. B) It forces React to unmount/recreate elements on every render. C) It is insecure.
```
<details> <summary style="cursor: pointer; color: #facc15; font-weight: bold;">â–¼ Click to Reveal Answer</summary>

âœ… Correct Answer: B

Rationale: Keys tell React which elements are stable. If the key changes every render, React thinks it's a completely new element, destroys the old DOM, and builds a new one. This kills performance and loses state (like input focus).

</details>

Q9: JS Sorting
```javascript

const arr = [1, 2, 15, 3];
arr.sort();
Result?
```
A) [1, 2, 3, 15] B) [1, 15, 2, 3] C) [15, 3, 2, 1]

<details> <summary style="cursor: pointer; color: #facc15; font-weight: bold;">â–¼ Click to Reveal Answer</summary>

âœ… Correct Answer: B

[1, 15, 2, 3]

Rationale: Default .sort() converts elements to strings and sorts alphabetically. "15" comes before "2". Use arr.sort((a,b) => a-b) for numbers.

</details>
