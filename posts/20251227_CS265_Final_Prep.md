### Q2: Types of Software Faults

**Task:** Explain three different types of faults that can occur in software systems.

<details>
<summary>â–¼ Show Answer</summary>

1. **Stress / Overload Faults**
   Occur when system load exceeds the specified maximum, causing incorrect behavior or failure.

2. **Capacity / Boundary Faults**
   Occur when system resources (storage, memory, limits) exceed defined capacity.

3. **Timing / Coordination Faults**
   Occur when concurrent processes fail to meet required execution order or timing constraints.
   Common in **real-time systems**.

ğŸ“Œ **Exam Insight:**
Naming + short explanation = full marks. Do not overwrite.

</details>

### Q3: Exhaustive Testing

**Task:** Define exhaustive testing and explain why it is impractical.

<details>
<summary>â–¼ Show Answer</summary>

**Exhaustive Testing**
Testing **every possible combination** of inputs and outputs in a program.

**Why it is impossible:**

* **Execution time explosion**
  Even a simple program can require billions of years to test fully.
* **Test design complexity**
  Each input requires a *known correct output*.

ğŸ“Œ **Conclusion:**
Exhaustive testing is **theoretically ideal but practically infeasible**.

</details>

### Q4: Dynamic Testing

**Task:** Define Dynamic Testing.

<details>
<summary>â–¼ Show Answer</summary>

**Dynamic Testing**
Testing that involves **executing the software** and observing its runtime behavior.

* Inputs vary over time
* Outputs are evaluated during execution

ğŸ“Œ **Contrast:**
Static testing = no execution
Dynamic testing = execution required

</details>
